Pluggable motion engine

The exact same icon markup, animated three different ways. Pick the engine per-icon with data-engine, or set a default for a whole subtree with data-ew-engine. CSS is the zero-dependency default; GSAP and Motion are loaded lazily only when used.

applied via <body data-ew-engine>

CSS engine

data-engine="css"

GSAP engine

data-engine="gsap" (lazy)

Motion engine

data-engine="motion" (lazy)
<!-- Vanilla: just declare the engine on the icon -->
<svg class="ew-icon" data-motion="pop" data-engine="gsap">…</svg>

<!-- Or default a whole region; per-icon data-engine still wins -->
<section data-ew-engine="motion"> …icons here… </section>

// React
import { EwIcon } from '@ewooral/icons/react';
<EwIcon name="star" engine="gsap" trigger="hover" />
